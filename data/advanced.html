⚡ Wi-Fi Killer — Deauthentication Attack
Hardware: any ESP32, ESP8266, or 802.11 adapter in monitor mode
EDU ONLY

How It Works

The 802.11 standard originally had no authentication for management frames. A deauthentication (deauth) frame tells a client "you are disconnected from this AP." An attacker can spoof these frames — using the AP's MAC as the source — and flood a client continuously. The client re-associates, gets deauthed again, and effectively loses connectivity. 802.11w (PMF) fixes this but most consumer APs still don't enforce it.

Attack Flow

  1. Discover target AP BSSID and operating channel
  2. Set adapter to the same channel
  3. Craft 802.11 Deauth frame (frame control 0xC0, reason code 7)
  4. Spoof source MAC = BSSID of target AP
  5. Send to broadcast or specific client MAC in a loop

Defense / Mitigation

✦Enable 802.11w / PMF (Protected Management Frames) on AP and clients
✦Use WPA3-SAE (mandates PMF)
✦WIDS (Wireless Intrusion Detection) alerts on deauth floods
✦5 GHz band is harder to jam due to shorter range
📡 Fake SSIDs — Beacon Flood & Evil Twin
Hardware: any 802.11 adapter with raw-frame injection (ESP32, alfa cards)
EDU ONLY

How It Works

Beacon Flood: Beacon frames are broadcast by APs to announce their existence. An attacker sends thousands of fake beacons per second with random SSIDs and BSSIDs. This overwhelms the client's Wi-Fi scanner list and can cause denial-of-service on the scanning subsystem of some devices.

Evil Twin: A more targeted variant — the attacker clones an existing AP's SSID and BSSID on the same or adjacent channel, often combined with deauth on the real AP. Clients connect to the attacker's AP and all traffic can be intercepted (MITM).

Evil Twin Flow

  1. Scan target: record SSID, BSSID, channel, security type
  2. Create AP with identical SSID on same channel
  3. Deauth clients from legitimate AP
  4. Clients auto-connect to evil twin
  5. Run DHCP + DNS / SSL-strip / capture credentials

Defense / Mitigation

✦Use certificate-pinned VPN over all Wi-Fi connections
✦HSTS / HSTS Preload prevents SSL stripping on HTTPS sites
✦802.1X / EAP enterprise auth (validates AP via PKI)
✦Rogue AP detection in corporate WIDS
✦WPA3 OWE / SAE hardens handshake against MITM
📻 HackRF One — RF Jamming & GPS Spoofing
Hardware: HackRF One (1 MHz – 6 GHz SDR, ~$300) + GNU Radio / GQRX
EDU ONLY

What Is HackRF?

HackRF One is an open-source Software-Defined Radio (SDR) capable of transmitting and receiving signals from 1 MHz to 6 GHz. Unlike the ESP32 which only handles 2.4 GHz Wi-Fi / BLE, HackRF can target any radio protocol in that range — GSM, LTE, DECT, LoRa, Z-Wave, 433 MHz remotes, GPS (L1 = 1575.42 MHz), and more.

RF Jamming Concept

  1. Identify target frequency and modulation (e.g. 433 MHz OOK)
  2. Generate wideband noise or continuous wave at that frequency
  3. Target receivers cannot distinguish signal from noise (SNR drops)
  4. Result: denial of service on radio channel (alarms, remotes, drones)

GPS Spoofing Concept

  1. GPS uses very weak signals from satellites (~-130 dBm at ground)
  2. HackRF generates fake GPS L1 signals at higher power than real satellites
  3. Target device locks onto fake signal — reports wrong location / time
  4. Applications: drone navigation, ship AIS, financial timestamps

Defense / Mitigation

✦GPS anti-spoofing chips (multi-constellation, signal-integrity checks)
✦Faraday caging sensitive RF receivers
✦RF anomaly detection (sudden SNR drop = jamming)
✦Jamming is illegal in almost every country (ITU Radio Regulations)
🐬 Flipper Zero — Sub-GHz Replay Attack
Hardware: Flipper Zero (300–928 MHz sub-GHz, NFC, RFID, IR, iButton)
EDU ONLY

What Is Flipper Zero?

Flipper Zero is a portable multi-tool for pentesters and hardware hackers. Its Sub-GHz radio (CC1101 chip) covers 300–928 MHz, letting it record and replay RF signals from garage doors, car key fobs, barrier gates, and wireless alarm sensors that use fixed codes (no rolling codes / challenge-response).

Sub-GHz Replay Flow

  1. Put Flipper into Sub-GHz read mode on common frequency (e.g. 433.92 MHz)
  2. Capture signal when victim presses garage-door remote
  3. Flipper records raw OOK/FSK modulation + timing
  4. Attacker replays the saved signal — door opens
  5. (Advanced) Rolljam: jam + record rolling code, replay later

Other Flipper Capabilities

  • NFC clone (MIFARE Classic with weak keys)
  • 125 kHz RFID badge emulation (HID, EM4100)
  • iButton / Dallas key read+emulate
  • IR universal remote database
  • BadUSB (keyboard injection payload)
  • BLE spam (Apple / Android popups)

Defense / Mitigation

✦Rolling-code systems (KeeLoq, AUT64) — each press uses a new code
✦UHF challenge-response for access control (mifare DESFire, SEOS)
✦Replace 125 kHz RFID with 13.56 MHz NFC with crypto
✦Faraday sleeve / signal-blocking wallet for cards
💥 Bluetooth Jamming
Hardware: HackRF / nRF24L01+ with amplifier / custom BT jammer
EDU ONLY

How Bluetooth Works vs. How Jamming Works

Classic Bluetooth and BLE operate in the 2.4 GHz ISM band (2.402–2.480 GHz), divided into 79 channels (Classic) or 40 channels (BLE). Bluetooth uses Adaptive Frequency Hopping (AFH) — it hops through channels 1600 times/second to avoid interference. Jamming must therefore either:

  • Wideband noise: saturate the entire 2.4 GHz band with noise (requires high power — affects Wi-Fi too)
  • Reactive jamming: detect BT preamble and immediately transmit interference on the same channel (lower power, harder to detect)
  • BLE advertisement jam: target only the 3 advertising channels (37, 38, 39) to prevent device discovery / pairing

Attack Scenarios

  1. Jam BLE medical device (insulin pump, heart monitor) — safety-critical
  2. Prevent BT keyboard / mouse pairing (DoS)
  3. Disrupt BLE-based smart locks / access control
  4. Suppress BLE beacons / asset-tracking tags (AirTags, Tile)
  5. Create RF noise to cover side-channel exfiltration on 2.4 GHz

Defense / Mitigation

✦BT 5.0 LE Coded PHY extends range — harder to jam at distance
✦Shielded environments (Faraday rooms) for critical BT devices
✦Wired fallback for medical / industrial devices
✦Spectrum analyzers to detect 2.4 GHz jamming events
✦Intentional jamming is a federal crime in most countries

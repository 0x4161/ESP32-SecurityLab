What is DNS Spoofing?

DNS maps domain names to IP addresses. An attacker on the same network can intercept DNS queries and reply with a forged IP, redirecting the victim to a malicious server before the real DNS responds.

⚠ Real DNS spoofing requires network access and is illegal without explicit authorization. Concept only.
Defences
  • Use DNSSEC to cryptographically verify responses
  • Enable DNS-over-HTTPS (DoH) or DNS-over-TLS (DoT)
  • Use a trusted resolver (1.1.1.1 / 8.8.8.8)
  • Enforce HTTPS + HSTS — cert mismatch reveals redirect
  • Monitor DNS traffic for anomalous TTLs or IPs

What is ARP Spoofing?

ARP resolves IP addresses to MAC addresses on a local network. Sending forged ARP replies lets an attacker associate their MAC with the gateway IP, placing themselves as a Man-in-the-Middle between victims and the network.

⚠ ARP attacks on unauthorized networks are a criminal offence. Educational demonstration only.
Defences
  • Dynamic ARP Inspection (DAI) on managed switches
  • Static ARP entries for gateways and critical hosts
  • XDR / IDS: detect duplicate MAC for same IP
  • Always use TLS — MitM cannot read encrypted traffic
  • VPN on any untrusted or shared network

What is an Evil Twin AP?

A rogue access point advertising the same SSID as a legitimate AP (often with higher signal). Devices that auto-connect allow the attacker to intercept all traffic in a Man-in-the-Middle position.

✗ Deploying rogue APs on unauthorized networks is illegal. This project does NOT implement this attack.
Defences
  • Use WPA3 — prevents downgrade and PMKID attacks
  • Disable auto-connect for public / open SSIDs
  • Verify BSSID of connected AP matches known hardware
  • Enterprise WPA2 with certificate pinning
  • Always use HTTPS and verify TLS certificates
  • Force-route sensitive traffic over VPN
